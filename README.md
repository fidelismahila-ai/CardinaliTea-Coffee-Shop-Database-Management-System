# CardinaliTea-Coffee-Shop-Database-Management-System
CardinaliTea is a relational SQL database project designed to simulate the operations of a modern coffee shop. The system models key business processes including employee management, shift scheduling, customer orders, inventory tracking, supplier relationships, menu management, and payment processing.

The project demonstrates database design principles such as normalization, entity relationships, primary and foreign key constraints, and junction tables to manage many-to-many relationships. The schema enables efficient storage and retrieval of operational data and supports business insights such as order tracking, employee assignments, and inventory usage.

This project was developed as part of a database systems course and represents a complete relational model for a café management system.
